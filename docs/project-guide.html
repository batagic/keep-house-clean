Redirect tại root — vì sao cần?

Frontend thật nằm trong code/. GitHub Pages publish từ root repo, nên URL công khai là …/keep-house-clean/ chứ không phải …/keep-house-clean/code/ làm “gốc” duy nhất.

  • Các file HTML mỏng tại root (index.html, nhat-ky.html, kho-qua.html, quy-doi.html, print.html, admin/login.html, …) dùng <meta http-equiv="refresh"> + link canonical trỏ sang code/….
  • Bookmark & link cũ (/keep-house-clean/nhat-ky.html) vẫn mở đúng trang sau khi tái cấu trúc.
  • Không nhân đôi assets — chỉ vài dòng redirect, một bản CSS/JS trong code/assets/.
  • Entry production chính thức: code/nhat-ky.html (API config trỏ đúng path GitHub Pages).

Migrations — hai thư mục, một schema

Tránh nhầm lẫn khi deploy hoặc dựng DB mới:

Thư mụcVai trò
code/kho-thoc-api/migrations/Nguồn chạy — migrate.js, Docker, VPS rsync API
docs/db/migrate/Bản sao operator — SQL thuần, psql -f, audit lịch sử
  • Developer: thêm NNN_mo_ta.sql trong code/kho-thoc-api/migrations/ → npm run migrate → bash scripts/sync-migrations.sh → commit cả hai.
  • Operator: dựng DB sạch từ docs/db/migrate/*.sql (theo thứ tự) hoặc node scripts/migrate.js trong container.
  • Chi tiết: db/README.md

Prototype — không deploy

HTML thử layout cũ nằm trong docs/prototypes/ (trước đây code/ideas/). Không liên quan GitHub Pages hay API production — chỉ tham khảo thiết kế.

Quyết định quan trọng

  • Frontend tĩnh trên GitHub Pages — không build step
  • API Node.js Docker trên VPS, DB Postgres riêng kho_thoc
  • Migrate SQL — nguồn chạy: code/kho-thoc-api/migrations/; bản sao operator: docs/db/migrate/ (db/README.md)
  • Cách ly gia đình qua X-Family-Id sau phiên passcode (unlock_family)
  • Deploy: ./ops/vps/deploy.sh --all
